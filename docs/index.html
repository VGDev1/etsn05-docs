Frontend

Frontend

Carpool Frontend Client

Features
  • Vite with React, TypeScript and absolute imports.
  • Tailwind CSS v3 with a basic reset for form styles and a Prettier plugin that automatically sorts classes.
  • Use ESLint, stylelint and Prettier on VSCode and before you commit with Husky and lint-staged.
  • Write unit and integration tests with Vitest and Testing Library.
  • Write e2e tests with Cypress.

Getting started

Preqrequisites

This project is based on React and TypeScript. Therefore you need to have Node.js installed.

  • If a package manager is used to install Node.js, it is recommended to follow the instructions on the Node.js website.
  • For binaries, e.g when using Windows: Node.js v16 [LTS]

Node.js naturally installs a package manager, npm - which will be used for installing our package-manager pnpm.

For Installing pnpm, run the following in your terminal of choice:

npm install -g pnpm

Make sure to cd into the frontend folder before running the following commands.

Install the dependencies:

pnpm install

After installing all dependencies you are good to go.

To simply run the program and try it out, run the following command:

pnpm dev

This will start the development server and open a browser window with the application running.

Extensions for VSCode (Optional)

Here follows a list of extensions that are recommended to be installed in VSCode for a better development-experience.

  • Reload - Adds a reload button to VsCode that reloads the editor, which is quite convenient when changes are made to configs and other caching issues.
  • Stylelint - Adds linting for CSS and SCSS files.
  • Rainbow Brackets - Adds color to brackets, parentheses, etc. to make it easier to see where they start and end.
  • Tailwind CSS IntelliSense - Adds autocomplete for Tailwind CSS classes. Useful as we use Tailwind for styling.
  • ESLint - Adds linting for JavaScript and TypeScript files. Useful for avoiding ESLint errors as we have configured an "aggressive" linter in order to preserve a code standard.
  • Better Comments - Adds syntax highlighting for comments. Useful for adding comments that are easier to read, with more options than normal comments. Note that these are only visible in VsCode, and for others that have not installed the extension, the comments will be normal.
  • ES7+ React/Redux/GraphQL/React-Native snippets - Adds snippets for React. Useful for quickly writing React code.
  • Prettier - Adds formatting for JavaScript, TypeScript, CSS, SCSS, JSON, Markdown, YAML, and GraphQL files. Useful for formatting code to a standard. -Github Copilot - Adds a code assistant that can write code for you. Useful for quickly writing code, but not recommended to use as a replacement for writing code yourself.

Themes that are very useful for making your environment more pleasant to work in:

  • One Dark Pro - A dark theme that is very pleasant to work in.
  • Material Icon Theme - Adds icons to files and folders. Useful for quickly identifying files and folders.

Scripts

Here is a list of scripts that can be run within the project, followed by a description of what they do.

  • pnpm dev - start a development server with hot reload. To be used when developing for continous updates in realtime when changes are saved.
  • pnpm build - Generates a build for production. The generated files will be on the dist folder.
  • pnpm preview - locally preview the production build.
  • pnpm test - run unit and integration tests related to changed files based on git.
  • pnpm test:ci - run all unit and integration tests in CI mode.
  • pnpm test:e2e - run all e2e tests with the Cypress Test Runner.
  • pnpm test:e2e:headless - run all e2e tests headlessly.
  • pnpm format - format all files with Prettier.
  • pnpm lint - runs TypeScript, ESLint and Stylelint validation. Useful for checking if there are any errors in the code and formatting numerous files at once.
  • pnpm validate - runs lint, test:ci and test:e2e:ci.

Contributing

If you intend to change the Client or UI, or make any non-trivial changes to the implementation, it is recommended to file an issue. This lets us reach an agreement on your proposal before you put significant effort into it, however please verify that there are no open issues related to your proposal.

If the fix is related to a bug, please include a minimal reproducible example in the issue if possible. This makes it easier to verify that the bug was fixed. It is also valuable for archiving purposes if a bug similar to this appears in the future.

Branches

The frontend branch is in a releasable state at all times, thus should never be commited to directly unless small changes (config, docs). We should be able to release a new minor version from the tip of main at any time.

Changes should be committed to an issue specific branch. Branches containing a feature and/or solving a sumbitted issue should always be created from the latest version of frontend.

Style guide

We use an automatic code formatter called Prettier. Run pnpm format in the terminal after making any changes to the code.

Then, our linter(s) will catch most issues that may exist in your code and try to fix them automatically. You can check the status of your code styling by simply running pnpm lint.

When commiting code, we use husky as a pre-commit hook to run the linter and formatter. If there are any errors, the commit will be aborted. If there are any warnings, the commit will be allowed, but you should fix the warnings before pushing. All errors and warnings will be printed to the terminal with information regarding what file the error occurs in, what error rule you are violating followed by an informative message. Further information regarding what command that failed is also printed to the terminal as "ERROR: "run-x" exited with y", when x refers to the specific run-command that failed and y refers to the exit code of the command.

For more info on what scripts are available for linting and what the involve, see the Scripts section.

However, there are still some styles that the linter cannot pick up. If you are unsure about something, looking at Airbnb’s Style Guide will guide you in the right direction.
